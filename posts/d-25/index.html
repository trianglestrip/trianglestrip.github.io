本文档把原文技术点拆成可执行清单，并对照当前 FireSmokeTest 的 Godot 实现标注状态。

状态说明：

  • 已实现：当前 Godot 项目已有对应机制，且运行路径已接入。
  • 部分实现：已有近似实现，但与文章描述仍有差距。
  • 未实现：当前没有对应功能。
  • 不计划：因为 Godot demo 范围、性能或缺少游戏对象系统，暂不实现。

1. 核心思想

技术点原文描述Godot 对照状态
使用实时 2D 流体模拟驱动火焰不依赖传统粒子系统，火焰、烟、运动都来自共享流体场新增 scripts/ignitement_fluid_2d.gd，火焰模式使用独立 2D solver已实现
通过少量纹理模拟物理量使用 Density、Velocity、Temperature、Reaction 等纹理当前 solver 使用 velocity、density、temperature、reaction、pressure、divergence、curl、light_map已实现
使用 fragment shader / Graphics.Blit 风格 pass原文刻意不用 compute shader，依赖 Blit 和纹理采样Godot 使用 SubViewport + ColorRect + ShaderMaterial + RenderingServer.force_draw(false) 模拟 Blit pipeline已实现
高兼容性优先Fragment shader 比 compute shader 更兼容当前实现全部为 canvas item fragment shader已实现

2. Simulation

原文伪代码：

void Simulate(float dt)
{
    RenderEmittersAndObstaclesToTexture();
    ApplyDiffusion(dt);
    ApplyAdvection(dt);
    ApplyExtinguishmentImpulse(dt);
    ComputeVorticityConfinement(dt);
    ComputeDivergence();
    ComputePressure();
    ComputeProjection();
    AdvectParticles(dt);
}
技术点原文描述当前实现状态
Density 纹理烟雾和可见空气厚度，原文示例为 1024x1024 RGBA halfdensity_a/b，当前大小 512x288，RGBA viewport texture部分实现
Velocity 纹理控制流动方向和速度，原文示例为 512x512 RG halfvelocity_a/b，编码为 RG，0.5 表示零速度已实现
Temperature 纹理控制高温区域、浮力、烟雾/fog 解释temperature_a/b，单通道使用 .r已实现
Reaction 纹理火焰强度、燃料、传播行为reaction_a/b，.r 表示 flame，.g 表示 fuel已实现
Render emitters把火源、烟源、速度源写入纹理emit_fire() / emit_smoke()，由 main.gd 沿喷射方向多点调用已实现
Obstacles把世界障碍写入模拟场，影响速度边界旧 FluidSolver2D 有 player obstacle，新 IgnitementFluid2D 暂无 obstacle pass未实现
Diffusion流体扩散当前没有单独 diffusion pass，只通过 advection、dissipation、pressure/vorticity 近似未实现
Advection沿 velocity 场输运 velocity/density/temperature/reactionignite_advect.gdshader已实现
Extinguishment / fire to smoke火焰产生烟，reaction 转 densityignite_reaction.gdshader 中 flame 生成 density，temperature 冷却已实现
Vorticity confinement增强流体卷曲复用 fluid_curl.gdshader 和 fluid_vorticity.gdshader，CURL_STRENGTH := 76.0已实现
Divergence计算速度散度fluid_divergence.gdshader已实现
Pressure solveJacobi 迭代压力fluid_pressure.gdshader，PRESSURE_ITERATIONS := 34已实现
Projection减去压力梯度，使速度场近似无散fluid_gradient_subtract.gdshader已实现
Advect particlesGPU 粒子采样 velocity 场运动当前没有自定义 GPU particle buffer；旧 GPUParticles 已关闭未实现
Simulation domain follows camera模拟区域跟随摄像机移动，制造无限场假象当前 demo 固定 1280x720 屏幕域未实现

7. 未实现清单

优先级高：

  • 新 solver obstacle pass：让玩家、墙、箱子真正影响火/烟流动。
  • 环境对象采样 light map：让地面、箱子、角色被火光照亮。
  • 自定义 ember particles：粒子采样 velocity 场随流体漂移。

优先级中：

  • 单独 diffusion pass：更接近完整 Stable Fluids pipeline。
  • Temperature-driven smoke/fog 重构：让 smoke 更多从 temperature 解释，而不是 density 主导。
  • Reaction readback：为 burn/damage 或动态火焰碰撞做准备。

优先级低：

  • Simulation domain follows camera：当前 demo 是固定屏幕，不需要无限世界。
  • UI light map reflection：当前 demo UI 很简单，收益不高。
  • Vertex wiggle：没有对应环境材质系统。
